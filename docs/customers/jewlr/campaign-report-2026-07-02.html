Why only 23% of ticket order IDs match the purchase table:
Tickets from the last 30 days frequently reference old orders — ring resizing, repairs, returns, lifetime warranty claims. The purchase table starts May 2024, but matched orders skew heavily recent: 96% of matched tickets (1,546/1,618) have purchase dates in 2026. The remaining 5,269 unmatched ticket order IDs are almost certainly from orders placed before Stacktome's purchase data integration started.

Action Items
  • 1
    Fix order_id case mismatch in purchase table. Purchase table stores order IDs lowercase (e.g. j0108702286) while Stamped reviews and ticket subjects use uppercase (J0108702286). All joins need lower() normalization or the table should be standardized on import. This was causing a 4× undercount in ticket order matching.
  • 2
    Extend purchase data history beyond May 2024. 77% of ticket order IDs (5,269) can't be matched because they reference pre-2024 orders. For tickets about warranty, resizing, or repairs, this means Stacktome has no order context to enrich the ticket. If Jewlr can provide a historical order export, matching coverage for tickets would improve dramatically.
  • 3
    Investigate 9% of Stamped reviews missing order_id. 6 of 68 Stamped reviews have no order_id in the record. Check if these are anonymous (guest checkout) reviews in Stamped or a mapping gap in the Stamped integration. Email_hash is still available for manual matching.
  • 4
    Extract order_id from ticket subjects systematically. 81% of HelpScout ticket subjects contain a parseable Jewlr order ID. This regex extraction (J\d{10}) should be applied at ingestion time so that the order_id field is populated for all tickets (not just the 3% currently populated). This would improve all downstream enrichment.
  • 5
    Address recurring quality issues — stones falling out, material concerns. 3 reviews mention stones falling out within days/2 weeks; 1 flags plating sold as silver. These drive both Trustpilot 1★ reviews and HelpScout tickets. A QC check or product-specific follow-up message for affected SKUs (e.g. JWL0004, JWLN0300) could reduce both complaint channels simultaneously.
  • 6
    Improve CS response time for disputed quality claims. Multiple reviewers mention emailing CS with no response. The 149 currently active tickets and any ticket tagged with keywords like "quality", "stone", "silver" should be prioritized for reply.
